## âœ… Architecture Principles

| Principle | Implementation |
|-----------|----------------|
| **Separation of Concerns** | Each layer has one job |
| **Single Source of Truth** | Types defined once in `/types` |
| **Clean Imports** | `@/` alias for cross-folder imports |
| **Server vs Client** | Server Components for data fetching, Client for interactivity |
| **Reusable UI** | UI components in `_components/ui/` have no business logic |
| **Error Handling** | Services throw errors, hooks catch and show notifications |
